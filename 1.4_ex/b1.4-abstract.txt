
- Utilizar o comando find ;
	find [directory] -[name , date , size e type] [keyword] = pesquisa em diretorios por arquivos ou pastas
		-name = declarar "[keyword]" entre "", lista arquivo e diretorios
		-type [d, f] = d-diretorios
	history = lista historico de comando do terminal
		| [head, tail] = delimitar secao 
	echo "[msg]" = funciona como um console.log
		> [fileName] = funciona como um [cat] , inserindo mensagem no arquivo   

- Utilizar os operadores > , >> e | no terminal;
	> = input
	>> = anexa 
	< = output
	| = "pipe"
		cat > [fileName] = gravar conteudo no arquivo (caso o arquivo alvo ja tenha algum conteudo, esse sera sobrescrito)
			[ctrl] + D = encerrar input
		cat [fileName1] [fileName2] (... [fileNameN]) > [newFileName] = junta o conteudo de n arquivos num outro arquivo, na ordem em que os arquivos sao listados no comando
		cat >> [fileName] = acrescentar conteudo (append)
		sort [fileName] = ordena o conteudo do arquivo (somente visualizacao)

- Alterar as permissões de arquivos e diretórios;
	who = saber quem esta utilizando o sistema
	ls -l OU ll =  lista arquivo num formato "longo" 
		example:
  Permissões | Links  | Proprietário | Grupo | Tamanho | Data e Hora   | Nome
  -----------|--------|--------------|-------|---------|---------------|---------
  drwxr-xr-x | 2      | root         | root  | 4096    | Out 19 09:10  | composer/
	
		Permissões => É possível verificar o tipo do item e nível de permissão para Leitura, Escrita e Execução de item ou diretório;
		Links => Número de ligações que o item possui, no caso do diretório, número de subdiretórios que possui;
		Proprietário => Quem é o dono, quem criou. É o diretório padrão do usuário, o home;
		Grupo => Grupo ao qual pertence o item ou diretório. Utilizado para dar permissões à outras pessoas;
		Tamanho => Em Bytes;
		Data e Hora => Momento em que foi criado ou última modificação;
		Nome => Nome do item ou diretório;

	permissoes:
	[-][---][---][---] *padrao de exibicao 
	(tipo)(usuario)(grupo)(outros usuarios)
		r OU 4 = (read) somente leitura  
		w OU 2 = (write) editar o arquivo 
		x OU 1 = (execute) acessar/ executar 
		- = arquivo *no comeco da sequencia
		d = diretorio *no comeco da sequencia
		
	chmod [u, g, o, a][+, -, =][r, w, x] [fileName] = alterar permissoes
		u = usuario
		g = grupo
		o = outros usuarios
		a = todos 
			- = retirar permissao 
			+ = acrescentar permissao 
			= = delimitar permissoes 
	**modo octal:
	chmod [nUser][nGroup][nOutrosUsuarios] [fileName]
		rwx = 111 ( 7 | Acesso Total )
		rw- = 110 ( 6 | Somente Leitura e Escrita )
		r-x = 101 ( 5 | Somente Leitura e Execução )
		r-- = 100 ( 4 | Somente Leitura )
		-wx = 011 ( 3 | Somente Escrita e Execução )
		-w- = 010 ( 2 | Somente Escrita )
		--x = 001 ( 1 | Somente Execução )
		--- = 000 ( 0 | Todos Acessos Negados )
	
- Verificar o status e enviar sinais a processos UNIX;
	ps = lista processos
		PID = (process ID) identificacao de processo
	sleep [n] = parar terminal durante n segundos
		ctrl+z = suspende processo atual
			bg = mover processo suspenso para background
			fg %[nDoProcesso] = retomar processo suspenso (foreground)
		ctrl+c = encerra processo 
	& = executar processo em background
	jobs = lista processos em execucao (suspenso e background tambem)
	kill (%[nDoProcesso] || [PID]) = encerrar processo
		-9 = force 

REALIZACOES
	-um arquivo nunca se modifica. qualquer "modificacao" e fruto da gravacao de um NOVO arquivo (com mesmo nome), com novos dados e a consequente *eliminacao do arquivo no estado anterior*
